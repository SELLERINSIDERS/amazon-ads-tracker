# Plan 06-02: Safety Limits Configuration

## Goal
Create settings UI for safety limits and validation service.

## Requirements Covered
- SAFE-01: Configure safety limits in settings
- SAFE-02: Enforce limits on action endpoints
- SETT-02: Safety limits settings

## Tasks

### 1. Create Safety Limits Settings Component
- Form for configuring all limits
- Input validation
- Save action

### 2. Create Safety Limits Service (lib/safety.ts)
```typescript
export async function getSafetyLimits(): Promise<SafetyLimit>
export async function updateSafetyLimits(limits: Partial<SafetyLimit>): Promise<void>
export function validateBidChange(currentBid: number, newBid: number, limits: SafetyLimit): ValidationResult
export function validateBudgetChange(currentBudget: number, newBudget: number, limits: SafetyLimit): ValidationResult
```

### 3. Add Safety Limits Section to Settings Page
- Below Amazon connection section
- Expandable card for limits configuration

### 4. Create Server Actions for Limits
- getSafetyLimitsAction
- updateSafetyLimitsAction

## Form Fields
- Max Bid Change % (default: 50)
- Max Budget Change % (default: 100)
- Max Daily Spend (optional)
- Min Bid Floor (default: $0.02)
- Max Bid Ceiling (default: $100)
